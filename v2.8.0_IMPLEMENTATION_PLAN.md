# v2.8.0 Implementation Plan: The "Architectural Purity" Reorganization

## 🎯 Goal
Strictly separate responsibilities across namespaces to achieve **SRP (Single Responsibility Principle)** and **Cognitive Simplification**. The CLI will be divided into **Action Namespaces** (Item, Collection, SLR) and **Reporting Namespaces** (Report, SLR Report).

---

## 1. Namespace: `item` (The Bibliographic Wrapper)
**Responsibility:** Direct CRUD and asset manipulation of Zotero bibliographic records.

- [ ] **Purge SLR Leakage:** 
    - Remove flags from `item list`: `--included`, `--excluded`, `--criteria`, `--persona`, `--phase`.
    - These filters move exclusively to `slr list`.
- [ ] **Expand `item inspect`:**
    - Allow comma-separated keys: `item inspect K1,K2,K3`.
    - Support key-file input: `item inspect --file keys.txt`.
    - This allows for the **Removal of `slr lookup`**.
- [ ] **Unified Purge:**
    - Consolidate `item pdf strip` into `item purge --files`.
    - Ensure `item purge` is the atomic standard for asset removal.

## 2. Namespace: `collection` (The Folder Wrapper)
**Responsibility:** Structural management of the Zotero hierarchy and mass operations on folder sets.

- [ ] **Remove Redundancy:**
    - Delete `collection pdf fetch`. Users must use `item pdf fetch --collection`.
- [ ] **Relocate Analysis:**
    - Move `collection duplicates` to `report duplicates`.
- [ ] **Relocate Quality Audit:**
    - Move "Folder Metadata Audit" (currently `slr verify --collection`) to `report audit`.

## 3. Namespace: `slr` (The Process Conductor)
**Responsibility:** Managing the Systematic Literature Review lifecycle and recording funnel decisions.

- [ ] **Delete Obsolete:**
    - Remove `slr lookup` (Replaced by `item inspect`).
    - Remove `slr verify` (Functionality moved to `report`).
- [ ] **Consolidate SDB Maintenance (`slr sdb`):**
    - Group all note-manipulation tools under a single management command:
        - `slr sdb upgrade` (was `slr migrate`)
        - `slr sdb reset` (was `slr reset`)
        - `slr sdb export` (was `slr sync-csv`)
- [ ] **NEW Sub-Namespace: `slr source` (The Search Ingester):**
    - **`slr source init`**: 
        - Creates the 4-phase folder structure for a new source (e.g., `raw_acm` -> `01_title_abstract`, etc.).
        - Provides interactive guidance on Zotero Group setup for isolation/collaboration.
    - **`slr source add`**: 
        - Targeted ingestion of search results (CSV/BibTeX/RIS) directly into the `raw_` collection.
        - Automatically handles "Canonical" normalization if needed.
    - **`slr source list`**: Lists all active SLR sources and their current health/counts.
- [ ] **Refine Analysis:**
    - Move diagnostic/visual tools to the `slr report` subcommand.

## 4. Namespace: `import` (The Bibliographic Ingester)
**Responsibility:** General-purpose, one-off bibliographic ingestion.

- [ ] **Decouple from SLR:**
    - `import file` remains for general RIS/BibTeX ingestion into ANY collection.
    - `import arxiv`, `import doi`, and `import manual` are preserved as atomic bibliographic tools.
    - SLR-specific mass ingestion moves to `slr source add`.

## 5. NEW Namespace: `report` (The Library Informant)
**Responsibility:** Analytical insights for general Zotero users (Non-SLR specific).

- [ ] **`report verify-latex`**: (From `slr verify --latex`).
- [ ] **`report audit`**: (From `slr verify --collection`). Validates missing Abstracts, DOIs, etc.
- [ ] **`report duplicates`**: (From `collection duplicates`).
- [ ] **`report stats` (New)**: Global library metrics (year distribution, item types).
- [ ] **`report attachments` (New)**: Disk usage and missing file report.

## 5. NEW Namespace: `slr report` (The Funnel Analyst)
**Responsibility:** Aggregating and visualizing SLR conduct and results.

- [ ] **`slr report status`**: (From `slr status`). Funnel dashboard.
- [ ] **`slr report prisma`**: (From `report prisma`). Generate Mermaid/JSON flowchart.
- [ ] **`slr report shift`**: (From `slr shift`). Snapshot comparison.
- [ ] **`slr report graph`**: (From `slr graph`). included-paper citation network.
- [ ] **`slr report exclusion-summary` (New)**: Breakdown of rejection reasons.
- [ ] **`slr report consensus` (New)**: Multi-reviewer discrepancy report.

---

## Implementation Sequence
1.  **Phase A: Infrastructure** - Create `report_cmd.py` and `slr/report_cmd.py` skeletons.
2.  **Phase B: Transfers** - Move existing logic (Verify, Status, Shift, Graph) to new homes.
3.  **Phase C: Cleaning** - Purge overlapping flags and commands from `item`, `collection`, and `slr`.
4.  **Phase D: Expansions** - Implement multi-key `inspect` and new reporting verbs.
5.  **Phase E: Validation** - Update the Hexa-Gate tests to reflect the new namespace structure.
